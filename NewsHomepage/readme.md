# News Homepage

A responsive and accessible **News Homepage layout** built with modern frontend practices.

Built with **React, HTML, and CSS/Tailwind**, focusing on **semantic structure, responsive design, and accessibility improvements (ARIA + keyboard-friendly navigation)**.

This project is a solution to a **Frontend Mentor challenge**.

## Features

### Layout Structure

- Hero section with featured article
- Sidebar section for latest news articles
- Bottom news list with ranked articles

---

### Navigation

- Responsive navbar with mobile menu
- Overlay-based sidebar menu
- Click outside to close mobile menu
- Clean navigation structure using semantic elements

---


### User Experience

- Hover states for interactive elements
- Clear visual hierarchy for readability
- Mobile-first layout behavior

---

### Responsive Design

- Mobile-first approach
- Flexible grid layout for desktop and mobile
- Adaptive typography and spacing



## Technologies Used

### React / JavaScript

- Component-based UI structure

- Event handling (`onClick`, state toggling)

---

### HTML5

- Semantic layout structure
- Accessible navigation patterns
- Proper content hierarchy (headings structure)

### CSS / Tailwind CSS

## What I Learned

- Structuring a multi-section homepage layout

## Reflection

This project improved my understanding of **real-world UI layout structure**, especially how large landing pages are broken into semantic and reusable sections.

Implementing a mobile navigation system and maintaining accessibility standards helped reinforce the importance of **state-driven UI + ARIA synchronization**.

Separating layout concerns (hero, sidebar, news list) made the codebase easier to scale and maintain.
